# v1.2.0-preview1

> **Preview Release** · June 18, 2024 · [NuGet](https://www.nuget.org/packages/WinUI.TableView/1.2.0-preview1) · [GitHub Release](https://github.com/w-ahmad/WinUI.TableView/releases/tag/v1.2.0-preview1)

## What's New

- Introduced `MinColumnWidth` and `MaxColumnWidth` properties in `TableView` (can be overridden per column via `MinWidth` and `MaxWidth`)
- Added `Visibility` property for columns to hide or show columns at runtime
- Added new properties in `TableViewComboBoxColumn`:
  - `IsEditable` — makes the `ComboBox` editable
  - `TextBinding` — allows binding to `ComboBox.Text`
  - `SelectedValueBinding` — allows binding to `ComboBox.SelectedValue`
- Implemented column auto-sizing using `GridLength` as `Width` type. Default column width is now `GridLength.Auto`
- Implemented cell selection support so users can select individual cells
- Added new `SelectionUnit` property:
  - `CellOrRow` (Default) — user can select rows or cells
  - `Cell` — user can only select cells
  - `Row` — user can only select rows
- Exposed `GenerateElement` and `GenerateEditingElement` for columns

## Breaking Changes

- Setting column widths from code-behind requires adjustments to use `GridLength`
- Methods, events, and event argument classes have been renamed:

| Old Name | New Name |
|----------|----------|
| `GetSelectedRowsContent` | `GetSelectedContent` |
| `GetAllRowsContent` | `GetAllContent` |
| `OnExportSelectedRowsContent` | `OnExportSelectedContent` |
| `OnExportAllRowsContent` | `OnExportAllContent` |
| `ExportSelectedRowsContent` | `ExportSelectedContent` |
| `ExportAllRowsContent` | `ExportAllContent` |
| `TableViewExportRowsContentEventArgs` | `TableViewExportContentEventArgs` |

## Known Issues

- Duplicate items in `ItemsSource` can lead to unexpected cell or row navigation behavior
- When the column width is too narrow for the cell's desired width, there may be layout issues with the right edge appearing clipped
- When a filter is applied to any column, cell selection may not function correctly
