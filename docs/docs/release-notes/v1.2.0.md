# v1.2.0

> **Stable Release** · August 14, 2024 · [NuGet](https://www.nuget.org/packages/WinUI.TableView/1.2.0) · [GitHub Release](https://github.com/w-ahmad/WinUI.TableView/releases/tag/v1.2.0)

## What's New

- Introduced `CellTemplateSelector` and `EditingTemplateSelector` properties in `TableViewTemplateColumn`
- Introduced `MinColumnWidth` and `MaxColumnWidth` properties in `TableView` to set min/max widths for all columns (can be overridden per column via `MinWidth` and `MaxWidth`)
- Added `Visibility` property for columns to hide or show columns at runtime
- Added new properties in `TableViewComboBoxColumn`:
  - `IsEditable` — makes the `ComboBox` editable
  - `TextBinding` — allows binding to `ComboBox.Text`
  - `SelectedValueBinding` — allows binding to `ComboBox.SelectedValue`
- Implemented column auto-sizing using `GridLength` as `Width` type. The default column width is now `GridLength.Auto`
- Implemented cell selection support so users can select individual cells
- Added new `SelectionUnit` property to control selection behavior:
  - `CellOrRow` (Default) — user can select rows or cells
  - `Cell` — user can only select cells
  - `Row` — user can only select rows
- Exposed `GenerateElement` and `GenerateEditingElement` for columns

## Bug Fixes

- Fixed an issue with keyboard shortcuts not working at startup [#25](https://github.com/w-ahmad/WinUI.TableView/issues/25)
- Fixed cell layout issue when size was narrower than the desired width of a cell
- Improved code to pick right column type when auto-generating columns
- Fixed an issue with cell width clamping to min/max widths
- Removed a separator at the end of the options flyout when export options are hidden

## Breaking Changes

- Setting column widths from code-behind requires adjustments to use `GridLength` as the width type
- Some methods, events, and event argument classes have been renamed:

| Old Name | New Name |
|----------|----------|
| `GetSelectedRowsContent` | `GetSelectedContent` |
| `GetAllRowsContent` | `GetAllContent` |
| `OnExportSelectedRowsContent` | `OnExportSelectedContent` |
| `OnExportAllRowsContent` | `OnExportAllContent` |
| `ExportSelectedRowsContent` | `ExportSelectedContent` |
| `ExportAllRowsContent` | `ExportAllContent` |
| `TableViewExportRowsContentEventArgs` | `TableViewExportContentEventArgs` |

## Known Issues

- Duplicate items in `ItemsSource` can lead to unexpected cell or row navigation behavior. See [microsoft/microsoft-ui-xaml#9803](https://github.com/microsoft/microsoft-ui-xaml/discussions/9803)
