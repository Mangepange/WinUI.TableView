# v1.3.0

> **Stable Release** · April 24, 2025 · [NuGet](https://www.nuget.org/packages/WinUI.TableView/1.3.0) · [GitHub Release](https://github.com/w-ahmad/WinUI.TableView/releases/tag/v1.3.0)

## 🌟 Highlights

- Upgraded to .NET 8
- Added Uno Platform Support
- Localization Support
- GridLines support
- Cell & Row Context Flyout
- Custom Sorting and Filtering

## What's New

- **Localization support**: added localization support to the control. Thanks to [@VisualAlf](https://github.com/VisualAlf) for de-DE translations and [@arthurits](https://github.com/arthurits) for es-ES translations. [#113](https://github.com/w-ahmad/WinUI.TableView/pull/113), [#116](https://github.com/w-ahmad/WinUI.TableView/pull/116), [#126](https://github.com/w-ahmad/WinUI.TableView/pull/126)
- **Gridlines**: Added support for customizable horizontal and vertical gridlines. These gridlines are enabled by default but can be turned off using `HeaderGridLinesVisibility` and `GridLinesVisibility` properties. Related properties: `VerticalGridLinesStrokeThickness`, `HorizontalGridLinesStrokeThickness`, `VerticalGridLinesStroke`, `HorizontalGridLinesStroke`. [#74](https://github.com/w-ahmad/WinUI.TableView/pull/74)
- **Alternate Row Colors**: Added support for alternate row background and foreground colors via `AlternateRowBackground` and `AlternateRowForeground` properties. [#83](https://github.com/w-ahmad/WinUI.TableView/pull/83), [#21](https://github.com/w-ahmad/WinUI.TableView/pull/21)
- **Cell & Row Context Flyout**: Added support for context flyouts on cells and rows, enabling custom actions via right-click menus. Set via `CellContextFlyout` and `RowContextFlyout`. [#85](https://github.com/w-ahmad/WinUI.TableView/pull/85), [#38](https://github.com/w-ahmad/WinUI.TableView/pull/38)
- **Custom Sorting and Filtering**: Implemented `IColumnFilterHandler` and introduced `Sorting` and `ClearSorting` events for custom data manipulation logic. [#98](https://github.com/w-ahmad/WinUI.TableView/pull/98), [#30](https://github.com/w-ahmad/WinUI.TableView/pull/30), [#55](https://github.com/w-ahmad/WinUI.TableView/pull/55), [#77](https://github.com/w-ahmad/WinUI.TableView/pull/77), [#92](https://github.com/w-ahmad/WinUI.TableView/pull/92)
- **Three-State Column Sorting**: Enhanced sorting functionality to include three states (Ascending, Descending, and None). [#98](https://github.com/w-ahmad/WinUI.TableView/pull/98), [#37](https://github.com/w-ahmad/WinUI.TableView/pull/37)
- **Corner Button Mode**: Implemented `CornerButtonMode` to choose corner button functionality. `ShowOptionsButton` property has been replaced with `CornerButtonMode`. [#80](https://github.com/w-ahmad/WinUI.TableView/pull/80), [#23](https://github.com/w-ahmad/WinUI.TableView/pull/23)
- **Tag Property**: `Tag` property added to the `TableViewColumn` type. [#84](https://github.com/w-ahmad/WinUI.TableView/pull/84), [#49](https://github.com/w-ahmad/WinUI.TableView/pull/49)
- **Customize Column Headers**: Column headers style can be set through `TableView.ColumnHeaderStyle` and `TableViewColumn.HeaderStyle`
- **Customize Cell Styles**: Cell style can be set through `TableView.CellStyle` and `TableViewColumn.CellStyle`
- **Element & Editing Element Style**: `TableViewBoundColumn` elements and editing elements style can be set through `ElementStyle` and `EditingElementStyle`
- Added `RowMinHeight` property [#110](https://github.com/w-ahmad/WinUI.TableView/pull/110)
- Added `PlaceholderText` property for `TimeColumn`
- Exposed `ScrollRowIntoView` and `ScrollCellIntoView` methods [#123](https://github.com/w-ahmad/WinUI.TableView/pull/123)
- Ensure cells work when using different `ItemsPanel` other than `ItemsStackPanel`
- Disallow selection and option buttons while editing

## New Column Types

- **TableViewDateColumn** – Supports `DateTimeOffset`, `DateTime`, and `DateOnly`. Uses new `TableViewDatePicker` control designed to work with multiple data types. [#87](https://github.com/w-ahmad/WinUI.TableView/pull/87)
- **TableViewTimeColumn** – Supports `DateTimeOffset`, `DateTime`, `TimeOnly`, and `TimeSpan`. Uses new `TableViewTimePicker` control designed to work with multiple data types. [#87](https://github.com/w-ahmad/WinUI.TableView/pull/87)

## What's Changed

- `AdvancedCollectionView` from Community Toolkit is replaced with custom type `CollectionView`. This change allows better handling of items and offers good feature integration with `TableView`. [#91](https://github.com/w-ahmad/WinUI.TableView/pull/91)

## Fixes

- Fixed select all check box fast click issue [#61](https://github.com/w-ahmad/WinUI.TableView/pull/61), [#81](https://github.com/w-ahmad/WinUI.TableView/pull/81)
- Crashes caused by empty property paths in column bindings are fixed [#42](https://github.com/w-ahmad/WinUI.TableView/pull/42), [#91](https://github.com/w-ahmad/WinUI.TableView/pull/91)
- Fixed a crash when there are null values in primitive type columns [#89](https://github.com/w-ahmad/WinUI.TableView/pull/89), [#91](https://github.com/w-ahmad/WinUI.TableView/pull/91)

## Breaking Changes

- `ShowOptionsButton` property has been removed in favor of `CornerButtonMode` property
- `CollectionView` property is now of type `ICollectionView`

## Known Issues

- Changing grid line thickness or visibility at runtime may lead to unexpected layouts of column headers and row cells. This is due to an issue in `ItemsStackPanel` [microsoft/microsoft-ui-xaml#9860](https://github.com/microsoft/microsoft-ui-xaml/issues/9860)
