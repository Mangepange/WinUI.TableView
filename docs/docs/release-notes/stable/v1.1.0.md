# v1.1.0

> **Stable Release** · May 5, 2024 · [NuGet](https://www.nuget.org/packages/WinUI.TableView/1.1.0) · [GitHub Release](https://github.com/w-ahmad/WinUI.TableView/releases/tag/v1.1.0)

## What's New

- In filter flyout, use `Contains` instead of `StartsWith` when searching (Thanks to [@XamlBrewer](https://github.com/XamlBrewer)) [#6](https://github.com/w-ahmad/WinUI.TableView/issues/6)
- Removed package dependencies:
  - [WinUIEx](https://www.nuget.org/packages/WinUIEx/) (Thanks to [@BreeceW](https://github.com/BreeceW)) [#1](https://github.com/w-ahmad/WinUI.TableView/issues/1)
  - [CommunityToolkit.WinUI.Controls.Sizers](https://www.nuget.org/packages/CommunityToolkit.WinUI.Controls.Sizers/)
  - [CommunityToolkit.WinUI.Converters](https://www.nuget.org/packages/CommunityToolkit.WinUI.Converters/)
  - [CommunityToolkit.WinUI.Helpers](https://www.nuget.org/packages/CommunityToolkit.WinUI.Helpers/)
- New properties introduced in this release:
  - `TableView.IsReadOnly` — default is `false`; `true` makes all columns read-only
  - `TableView.CanUserSort` — default is `true`; `false` makes all columns unsortable and clears existing sort
  - `TableView.CanUserFilter` — default is `true`; `false` makes all columns unfilterable and removes existing filters
  - `TableView.CanResizeColumns` — default is `true`; `false` makes all columns non-resizable
  - `TableView.ShowOptionsButton` — default is `true`; `false` hides the options button
  - `TableView.HeaderRowHeight` — default value is set to `48`

## Fixes

- Major performance improvement in handling newly added items
- Fixed column header layout issues on resize [#3](https://github.com/w-ahmad/WinUI.TableView/issues/3)
- `Select All` command enable state fixed based on selection mode [#4](https://github.com/w-ahmad/WinUI.TableView/issues/4)
- Fixed order of copy or export for selected items output [#5](https://github.com/w-ahmad/WinUI.TableView/issues/5)
- Fixed cell width bindings with their column
